# ForgeNote · 低保真线框规格

> 状态：❄️ **已冻结（2026-07-12，G0S-12 阶段 0）——随 CCOS 冻结；可复用骨架将折入 PHASE-2**  
> 文档版本：v1.0  
> 日期：2026-07-11

## 1. 文档目的

本文档把已定义的信息架构与 UX Flow 转化为可制作、可点击、可测试的低保真页面结构。

线框只表达：

- 信息层级
- 页面分区
- 主次操作
- 页面与阶段流转
- 关键状态和恢复入口

线框不表达品牌视觉、最终文案、颜色、字体、图标风格、动效或工程实现。

依据：

- `USER-JOURNEY.md`
- `INFORMATION-ARCHITECTURE.md`
- `UX-FLOW.md`
- `UX-STRUCTURE-VALIDATION.md`

## 2. 线框约定

### 2.1 标记

| 标记 | 含义 |
|---|---|
| `[主操作]` | 当前页面唯一强调操作 |
| `[次操作]` | 不与主操作竞争的操作 |
| `(选择项)` | 单选、筛选或阶段选择 |
| `{系统状态}` | 保存、生成、错误等反馈 |
| `→ P-xx` | 页面跳转 |
| `→ S-xx` | 同页状态切换 |

### 2.2 框架规则

- 桌面 Web 为 MVP 主视口，基准宽度 1440px。
- 主内容最大宽度建议 1120–1200px；编辑器优先获得空间。
- 全局导航位置在原型中保持一致。
- 每个页面最多一个视觉主操作。
- 内容工作页以同一任务上下文为核心，不拆成多个路由页面。
- 低保真测试使用灰阶，不用品牌色暗示正确路径。

### 2.3 共用应用框架

```text
┌──────────────────────────────────────────────────────────────────┐
│ ForgeNote  工作台  工作流  历史                [＋ 新建灵感] 账户 │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│                         页面内容区                               │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

## 4. P-02 灵感收集

### 4.1 默认状态 S-02A

```text
┌──────────────────────────────────────────────────────────────────┐
│ ← 返回工作台                                      {尚未保存}    │
├──────────────────────────────────────────────────────────────────┤
│ 记下一条灵感                                                     │
│ 不用整理完整，先保留原始想法。                                   │
│                                                                  │
│ ┌──────────────────────────────────────────────────────────────┐ │
│ │ 输入想法、摘录或粘贴链接……                                  │ │
│ │                                                              │ │
│ │                                                              │ │
│ └──────────────────────────────────────────────────────────────┘ │
│ 来源 / 链接（可选）                                               │
│ [____________________________________________________________]   │
│ 标签（可选） [________]                                          │
│                                                                  │
│ [仅暂存]                                      [保存并整理]       │
└──────────────────────────────────────────────────────────────────┘
```

### 4.2 输入不足 S-02B

在输入区下方局部显示：

```text
内容可以先暂存。如果希望立即整理，请补充你想讨论的主题或观点。
[仅暂存]   [继续补充]
```

不得把过短输入直接清空或表现为系统错误。

### 4.3 链接解析失败 S-02C

```text
{无法读取这个链接，但链接已保留}
可以粘贴摘要帮助整理，或先保存稍后处理。
[先暂存] [补充摘要]
```

### 4.4 保存失败 S-02D

- 反馈位于操作区上方。
- 明确“输入仍保留在此页面”。
- 主操作切换为 `[重新保存]`。

### 4.5 页面去向

- 仅暂存 → P-01，并显示暂存成功反馈。
- 保存并整理 → P-03 / 整理阶段。
- 有未保存输入时返回 → 离开确认，不清空内容。

│ 上游依据 / 版本来源   │ 当前阶段工作区                                    │
│                      │                                                     │

### 5.2 框架规则

- 左区回答“结果基于什么”，右区回答“现在编辑什么”。

- 保存状态持续可见但不抢占主操作。
- 导出仅在存在可用版本时启用。

## 15. 关键确认与错误模式

### 15.1 离开未保存内容

```text
这部分修改尚未保存。
[继续编辑] [放弃修改]
```

若本地内容可恢复，应明确说明保存位置，不使用含糊的“确定离开？”。

### 15.2 AI 生成失败

必须回答：

- 哪个对象生成失败
- 已有内容是否安全
- 用户现在可重试、编辑还是退出

### 15.3 内容来源过期

使用非阻断提示，不把旧版本标为错误；由用户决定是否更新。

### 15.4 继续与复用确认

创建副本时显示：

```text
将基于这条内容创建一个新任务，原内容保持不变。
[取消] [创建新任务]
```

## 16. 响应式低保真原则

MVP 以桌面为主，仍需定义窄屏降级：

- 全局导航收起，但新建灵感保持可达。
- 内容工作页左侧依据区折叠为顶部抽屉。
- 阶段导航支持横向滚动，不压缩成不可读图标。
- 编辑器、主操作和保存状态优先保留。
- 低于可安全编辑宽度时，不并排展示多版本；改为标签切换。

移动端完整编辑体验不作为 MVP 验收重点，但不得造成内容无法访问或保存。

## 18. 进入视觉设计前的线框验收
