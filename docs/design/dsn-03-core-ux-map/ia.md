# DSN-03 · Information Architecture（信息架构 — 桌面 + 移动）

> 覆盖路由：`/login`、`/forge`、`/recipes`、`/recipes/[id]`、`/profile`，以及未来 history/performance 表面（M1 不独立成页）。
> `/forge` 北极星：左＝账号上下文，中＝当前任务 + 内容方案，右＝假设/控制/配方，底＝session/复用/表现。**桌面工作台不得拍平回单列长文档。**

## 0. 站点地图 Sitemap

```text
ForgeNote (M1)
├─ /login                          匿名入口（含同页注册、忘记密码流程）
│   └─ /login/reset (拟)           设置新密码回跳（DSN-02）
├─ /auth/callback                  OAuth / 邮件回跳（无 UI，纯处理）
├─ /forge                          工作台（引导态 ⇢ 工作台态，渐进显形）
│   └─ /forge?session=<id>         具体一次生成的持久化视图
├─ /recipes                        配方库（列表 / 搜索 / 类型筛选 / 删除）
├─ /recipes/[id]                   配方详情 + 换输入重跑
└─ /profile                        偏好（账号级假设默认值）
```

全局 chrome：顶部精简导航 `Forge · 配方库 · 偏好 · 退出`。**引导态新人**隐藏 配方库/偏好入口的视觉权重，避免空库劝退（渐进显形：存了第一个配方后再抬权重）。

## 1. `/login` IA

```text
桌面（分栏）                          移动（纵向）
┌───────────────┬───────────────┐    ┌─────────────────┐
│ 品牌区          │ 表单区          │    │ 吉祥物（缩小）      │
│ · ForgeNote 标  │ · 问候标题       │    │ 品牌标题           │
│ · 吉祥物演变小队 │ · 邮箱           │    │ 邮箱               │
│   (随光标/动效)  │ · 密码(显示切换)  │    │ 密码(显示切换)      │
│                │ · 记住30天/忘记   │    │ 记住30天 / 忘记     │
│                │ · 登录(主)        │    │ 登录(主)           │
│                │ · Google(次)      │    │ Google(次)         │
│                │ · 登录/注册切换    │    │ 登录/注册切换       │
└───────────────┴───────────────┘    └─────────────────┘
```

- 信息优先级：邮箱密码主路径 > Google 次 > 切换/恢复 文字级。
- 认证恢复子树：`忘记密码？` → 请求重置 → 重置邮件已发送 → 设新密码 → 完成 → 回登录（独立轻流程，不嵌主表单）。
- 吉祥物动效仅此页 + 空态允许（见 baseline 动效原则）。

## 2. `/forge` IA（核心，渐进显形）

### 引导态（新用户 · M1 主战场）

```text
桌面                                         移动
┌───────────────────────────────────────┐   ┌──────────────┐
│  你今天想做什么内容？（价值问句，非 hero 压迫）  │   │ 价值问句       │
│  ┌─────────────────────────────────┐   │   │ 大输入框        │
│  │ 大输入框（模糊想法）                  │   │   │ [可选]贴过往帖  │
│  └─────────────────────────────────┘   │   │ 示例任务 ×3     │
│  [可选] 贴一条你发过的帖（冷启动账号画像）      │   │ 「我会替你补」   │
│  示例任务：① ② ③                          │   │   标签预览       │
│  我会替你补这些： [受众][平台][形式][目标][风格] │   │ 先确认方向(主)   │
│  ───────────────────  先确认方向 (主操作)   │   └──────────────┘
└───────────────────────────────────────┘
（左/右/底空区此时不渲染）
```

### 工作台态（熟练用户 · 终态，逐块长出）

```text
桌面四区（北极星）
┌──────────┬───────────────────────┬──────────────┐
│ 左 账号上下文 │ 中 当前任务 + 内容方案        │ 右 假设/控制/配方 │
│ · 账号记忆   │ · 想法输入                  │ · 假设条(减字版)  │
│ · 内容资产   │ · 内容方案：正文/逐页卡片/    │ · 输出语言/偏好   │
│ · 已存上下文  │   配图方向/发布前检查        │ · 生成 / 重生成   │
│            │                          │ · 保存配方       │
├──────────┴───────────────────────┴──────────────┤
│ 底 连续性条：当前 session · 复用来源 · 表现回填入口            │
└────────────────────────────────────────────────┘

移动：单列优先级 → 价值/任务 → 输入 → 假设(折叠摘要) → 生成 → 内容方案 → 保存配方 → (底)session/表现
```

**显形触发**：存第一条账号记忆→左栏；有第一个历史 session→底部；存第一个配方→右栏配方区。连续生长、不跳变。

## 3. `/recipes` IA

```text
顶部：标题「配方库」+ 描述 · [新建配方→/forge]
工具条：搜索(按名) · 类型筛选 · 清空 · 「当前显示 n / 全部 n」
列表项：配方名 · 类型 · 更新时间 · 使用次数 · 来源session · [查看详情] · [删除(二次确认)]
空态：① 还没有配方 → 去 Forge 创建  ② 筛选无结果 → 换关键词/类型
移动：工具条折叠为单行 + 列表纵向卡片（信息降到 名称/类型/使用次数/详情）
```

## 4. `/recipes/[id]` IA

```text
返回配方库 ←
区块 A 元信息：创建/更新时间 · 使用次数 · 来源 session
区块 B 配方字段：受众 · 目标 · 语气 · 视觉规范 · 结构 · 可替换变量 · 禁止项 · 验收标准 · 版本
区块 C 换输入重跑：说明(沿用本配方/原配方不改) · 新主题输入(计数/上限) · [换输入重跑]
未找到态：配方不存在或不可见 → 回配方库
移动：A→B→C 纵向；B 字段两列降一列
```

## 5. `/profile` IA

```text
标题「偏好」+ 说明（下次 Forge 自动作为「来自偏好」假设带出）
新增偏好：内容类型 · 维度名 · 维度key · 偏好值 · [添加]（同 类型+key 覆盖）
列表：按类型分组 · 每条 维度/值/更新时间 · 行内编辑 · 删除
空态：还没有偏好 → 去 Forge 编辑假设后「记住为偏好」
移动：新增表单纵向堆叠；列表卡片化
```

## 6. 未来 history / performance 表面（M1 不独立成页）

- 表现回填寄生在 `/forge?session=` 的 OutcomePanel（最小入口），**不**新建 `/history`。
- 工作台底部「连续性条」是其长大归宿（渐进显形，M1 仅最小）。

---

## 导航与连续性规则（防患）

1. URL 即 session 身份：生成/草稿失败都写 `?session=`，刷新可恢复；新建清理旧 query。
2. 每页**一个主操作**：/login=登录，/forge 引导态=先确认方向、工作台态=生成内容方案，/recipes/[id]=换输入重跑。
3. 跨页连续：保存配方→详情→重跑→回 `/forge?session=`，结构化结果不退化。
4. 移动一律「优先级单列」降级，不复制桌面分区。
