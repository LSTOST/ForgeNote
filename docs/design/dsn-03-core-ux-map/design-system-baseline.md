# DSN-03 · Design System Baseline（设计语言基线）

> 不是 mood board，是**层级 + 规则**。目的：让设计语言**足够具体到能阻止下一次 login 式的局部 patch 螺旋**。
> 任何实现票必须能引用本文件某条规则；违反规则的 UI 一律 Codex 退回。

## 0. 决策层级（最高约束，按序）

```text
1. 工作台清晰 + 重复使用效率   ← 第一优先，永远赢
2. 暖纸感品牌氛围             ← 只能在不伤清晰度/对比度的前提下加
3. 少量灵动角色/动效（入口与空态）← 只在 1、2 都满足后点缀，绝不喧宾夺主
```

冲突时一律 1 > 2 > 3。**工作台屏永远密、可扫、安静**；角色动效不得与假设条/内容/配方控制争注意力。

## 1. 颜色角色 Color roles（角色制，不是色卡）

> 暖纸感承载氛围，但**正文/控件对比度必须达标**。**禁止一锅米黄/棕/橙**——用中性墨色做主文字、暖色只做品牌与强调。

| 角色 | 值 | 用途 |
|---|---|---|
| surface/base | `#FBF7F0` | 页面底（暖纸，浅） |
| surface/raised | `#FFFFFF` / `#FFFDF9` | 卡片、输入、面板 |
| brand/paper（仅 entry/空态） | `#F1E7D6` + 细点阵 | /login 品牌区、空态氛围；**不进工作台密集区** |
| ink/primary text | `#2A2320` | 主文字（高对比，非棕） |
| ink/secondary | `#6f6253` | 次要文字 |
| ink/muted | `#9a8e7c` | 辅助/占位 |
| accent/primary | `#B5562B` | 主按钮、主操作、品牌强调 |
| accent/hover | `#9F4924` | 主按钮 hover |
| border/line | `#E3D8C7` | 边框、分隔 |
| focus/ring | `rgba(181,86,43,.28)` 3px | 焦点环（统一暖色） |
| status/success | `#2F7A4F` | 成功 |
| status/danger | `#9E3322` / 底 `rgba(181,60,40,.07)` | 错误、危险 |
| status/info | `#5C6B4F`（暖中性）| 中性提示 |
| confidence/确定·推断·拿不准 | 实心 / 半 / 虚线描边（同 accent 族） | 假设条置信度，**用形态区分不堆新色** |

## 2. 字体层级 Type

- 字族：界面/正文 `Geist`（系统无则 system-ui）；品牌标题 `Newsreader`/本地 serif fallback（Georgia/Songti），**仅 /login 与空态品牌位**，不进工作台密集文本。**不引新远程字体**。
- 比例（桌面）：品牌标题 32–36 / 区块标题 18–20 / 子标题 15–16 / 正文 14–15 / 辅助 13 / 标签 12。移动正文 ≥15、输入 16（防 iOS 缩放）。
- 行高：标题 1.05–1.15；正文 1.5；密集列表 1.4。`text-wrap: pretty`。

## 3. 间距刻度 Spacing（4px 基）

```text
4 · 8 · 12 · 16 · 20 · 24 · 32 · 40 · 48
```
- 控件内边距 14–15；字段间 12；区块间 24；工作台分区 padding 24–32。
- 工作台密集区用**更紧的 12/16 节奏**保持可扫；入口/空态可放宽到 32/40 呼吸。

## 4. 按钮层级 Button hierarchy

| 级 | 样式 | 用途 | 规则 |
|---|---|---|---|
| primary | 实心 `#B5562B` / 字 `#FDF7EF` | 每屏唯一主操作 | 一屏**只有一个** |
| secondary | outline `#E3D8C7` / 字 `#6f6253` | Google、次操作 | 不与 primary 同视觉重量 |
| tertiary/text | 纯文字 accent 色 | 切换、忘记密码、记住为偏好 | 文字级，不做大按钮 |
| destructive | danger 描边/文字 + 二次确认 | 删除配方 | 必二次确认 |
- 高度 44（移动 46）、圆角 13、字 14–15、focus ring 统一。**禁止隐藏主操作**。

## 5. 输入 Input

- 高 44 / 圆角 13 / 背景 `#FFFDF9` / 边框 `#E3D8C7` / 字 15；focus：边框转 `#B5562B` + 3px ring。
- 标签优先**字段上方 label**（工作台/重置页）；登录可用占位但关键字段给 label。
- 密码：尾部显示/隐藏切换（`aria-pressed`）；「至少 8 位」仅注册/设密码态出现。
- 错误：字段下方内联 + 红边；表单级错误用 `role="alert"`。

## 6. 面板 / 区块 Panel & section

- 工作台分区用**安静的分隔 + 区块标题**（参考 Granola/Notion 高密度侧栏），**不要装饰性卡片堆**。
- 卡片仅用于「一个可操作对象」（配方项、内容方案块）；圆角 12–14、1px 暖边、极轻阴影。
- 假设条：**减字版** chip——摘要 + 置信度形态 + 「依据」点开；不是一排大卡片。
- 列表项（配方/历史）用**轻标签 + 行**（参考 Linear），不用大卡片。

## 7. 状态 / 错误 Status & error

- 成功：绿点/绿字轻提示，不抢屏。错误：暖红 alert（`role="alert"`），**永远带恢复操作**。
- 加载：骨架 + 安静进度文案；**不**用全屏 spinner 遮住已输入内容。
- 空态：插画/角色（仅此处允许）+ 一句「下一步去哪」+ 主操作。

## 8. 动效原则 Motion（强约束）

```text
允许：/login、onboarding、空态、轻量转场。
禁止：工作台密集区（假设条/内容方案/配方控制）出现角色/眼睛/装饰动效。
```
- 角色动效只服务「第一印象 + 空态有人情味」；**不得与任务控件争注意力**。
- 仅 `transform/opacity`、合成层；无首屏自动播放；指针跟随 rAF 节流。
- **`prefers-reduced-motion`**：关闭跟随/眨眼/跳晃/过渡，姿态瞬切；功能与可读性不依赖动效。
- 转场 ≤200ms ease-out；进度反馈优先「安静」而非炫。

## 9. 移动布局规则 Mobile

- 一律「**优先级单列**」降级，不复制桌面分区；桌面四区→移动按 任务 > 输入 > 假设(折叠) > 生成 > 结果 > 保存 > (底)表现 顺序。
- 触控目标 ≥44；输入字号 ≥16；左右安全边距 20。
- 无横向溢出、无横向滚动；标题不孤立单个汉字。

## 10. 反 patity 守则（写明「不要做什么」）

- ❌ 一锅米黄/棕/橙单色品牌；✅ 中性墨字 + 暖色强调。
- ❌ 装饰性卡片堆 / 大卡片列表；✅ 高密度行 + 轻标签。
- ❌ 隐藏或多个并列主操作；✅ 每屏唯一 primary。
- ❌ 把桌面工作台拍平成单列长文档；✅ 保持四区、移动才单列。
- ❌ 角色动效进工作台；✅ 仅入口/空态。
- ❌ 用户界面出现内部 taxonomy；✅ 用户向措辞。

> 这十条＋决策层级是「阻止下一次局部 patch」的硬约束：新票若触碰登录/工作台视觉，必须先指明遵守上面哪几条；做不到先回设计，不直接 patch。
