# DSN-03 · State Map（状态流）

> 每个状态必须写：**visible message（用户看见什么）· primary action · secondary action · recovery path（出错怎么回去）· data continuity（刷新/跨页数据是否还在）**。
> 文案为设计意图（最终中英进 `src/lib/copy`，去内部 taxonomy）。状态编号供实现票引用（如 `S-08`）。

## 横切状态规则

- **每个状态恰一个主操作**；secondary 不与 primary 抢视觉权重。
- **错误永远可恢复**：要么重试、要么回上一步、要么回上层页；绝不出现「死路 + 丢数据」。
- **数据连续是底线**：任何失败都不清空用户已输入的想法/假设/名称；可持久的都进 `?session=` / 库。
- **空态是引导，不是死页**：每个空态都给「下一步去哪」。
- 文案不外泄内部 taxonomy；登录失败不泄露账号是否存在。
