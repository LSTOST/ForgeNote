# G0S-12 · 单工作台外壳低保真线框（阶段 2 返工）

> 日期：2026-07-13  
> 唯一 IA 依据：`UX-BASE.md` §2.3「单工作台外壳」。  
> 范围：Owner 自用「独居生产台」，桌面优先、单语中文、小红书优先。本文只确定布局、信息层级、状态和交互，不确定颜色、字体、圆角、阴影或图标风格。

## 0. 线框约定

- `[主操作]`：当前视图唯一强调操作；`[次操作]`：不竞争的操作；`{状态}`：系统反馈。
- 登录后只有一个持久外壳：左栏常驻，主区随导航切换，左栏不重新加载。
- 实装采用 Next.js 共享 layout 并保留路由和深链，不做纯客户端 tab：写作 `/workspace`、选题 `/radar`、账号分析 `/account`、周看板 `/gate0`。
- 登录 `/login` 是外壳外唯一独立页。

## WF-S · 持久工作台外壳

```text
┌─────────────────┬──────────────────────────────────────────────────────┐
│ [⌂ Home] [◧ 收起左栏]                                                  │
│ [新写一条]      │                                                      │
│                 │                                                      │
│ 最近内容        │                主区视图容器                            │
│ · 内容 A        │     写作 / 选题 / 账号分析 / 周看板                    │
│ · 内容 B        │                                                      │
│                 │     仅主区替换；左栏状态、最近内容与账号菜单不重载      │
│ 本周可写选题    │                                                      │
│                 │                                                      │
│ 辅助            │                                                      │
│ · 周看板        │                                                      │
│ · 账号分析      │                                                      │
│                 │                                                      │
│ [头像 / 邮箱 ▴] │                                                      │
└─────────────────┴──────────────────────────────────────────────────────┘
```

| 状态 | 文案草稿 |
|---|---|
| 空态 | 「还没有最近内容，写第一条后会出现在这里。」 |
| 加载 | 「正在读取最近内容…」 |
| 错误 | 「最近内容暂时无法读取，主要入口仍可使用。」 |
| 成功 | 「已打开对应视图。」 |

**交互说明**

- Home 按钮切到默认写作视图；收起按钮折叠左侧工作区，再次点击可展开。
- 「新写一条」切到写作视图空状态并聚焦输入；最近内容切到写作视图并恢复任务。
- 「本周可写选题」「周看板」「账号分析」分别切换主区；当前项只有一个选中态。
- 视图切换保留左栏滚动、最近内容和账号菜单状态；浏览器后退按 URL 历史返回上一视图。
- 账号菜单仅「账号分析 / 退出登录」；点击外部或 Esc 关闭。

**与现状差异**

- 改：所有登录后视图归入同一个共享外壳，导航不随主区重建；`/workspace` 左上使用 Home 与收起左侧工作区按钮。
- 删：`/workspace` 产品 logo、独立 hub、辅助页顶栏、旧 dashboard 顶栏、重复身份。
- 不动：路由 URL、`?idea=` / `?taskId=` 深链、最近内容和账号菜单的数据职责。

### 工作流态（沿用 G0S-10）

```text
┌─────────────────┬──────────────────────────────────┬───────────────────┐
│ 常驻左栏        │ 中区 max 760                     │ 右栏 296          │
│                 │ 当前内容标题      {草稿保存状态} │ {唯一状态 Badge}  │
│                 │                                  │ 写作顺序          │
│                 │ Stage 0 想法输入                 │ 待确认            │
│                 │ [生成内容框架 · 主操作]          │ 账号分析摘要      │
│                 │            ↓                     │                   │
│                 │ Stage A 框架卡片                 │ Stage B 顺序只读  │
│                 │ [生成正文 · 主操作]              │                   │
│                 │            ↓                     │                   │
│                 │ Stage B 标题与分段正文           │                   │
│                 │ ── 产出区 ───────────────────── │                   │
│                 │ 小红书 [生成文案 + 卡片提示词]   │                   │
│                 │ 平台版本记录 → 查看 / 复制       │                   │
└─────────────────┴──────────────────────────────────┴───────────────────┘
```

| 状态 | 文案草稿 |
|---|---|
| 空态 | 「今天想写哪条内容？」 |
| 加载 | 「正在生成内容框架…」/「正在生成正文…」/「正在生成小红书版本…」 |
| 错误 | 「本次生成未完成，已有内容已保留，可直接重试。」 |
| 成功 | 「小红书文案与卡片提示词已加入版本记录。」 |

补充：未持久化「当前内容尚未保存，刷新后可能丢失」；部分失败「文案已保留，卡片提示词未生成，可单独重试」；会话过期「登录状态已过期，重新登录后继续已保存内容」。

**交互说明**

- 空状态提交后进入 Stage 0，输入保留；`Ctrl/Cmd + Enter` 仅在真实监听时可提交。
- 每阶段只显示一个主操作并紧跟内容；框架卡与右栏写作顺序双向 hover。
- 正文 1.2 秒防抖保存；产物记录常驻，查看层可用 Esc 关闭且记录不丢。
- 点击最近内容恢复想法、框架、草稿和产物；当前内容在左栏高亮。

**与现状差异**

- 改：原登录后首页输入并入写作空状态；写作视图不再有独立全站顶栏，标题和保存状态归中区头部。
- 删：独立首页、非中文/非小红书主流程、重复状态与身份。
- 不动：G0S-10 内容流、三区职责、唯一状态点和产出区。

│ 常驻左栏        │ 账号分析                                             │

| 状态 | 文案草稿 |

| 加载 | 「正在更新账号分析，原资料已保留…」 |

**与现状差异**

- 改：账号分析成为外壳内视图；左右明确「输入依据 / 生效判断」。
- 删：独立页面头、返回按钮和任何隐藏设置入口。
- 不动：资料输入、分析生成及其影响后续生成的职责。

## 7. 任务流覆盖

| UX-BASE 任务流 | 外壳内路径 |
|---|---|
| 新写一条 | 左栏「新写一条」→ 写作空状态 → Stage 0/A/B → 产出记录 → 复制 |
| 从选题开工 | 选题视图 → 选题卡 → 写作 Stage 0 预填 → 完成生产 |
| 重开继续 | 任意视图左栏最近内容 → 写作视图恢复 Stage B → 编辑/生成 |
| 发布后回填 | 产物查看层复制 → 左栏「周看板」→ 关联内容 → 保存记录 |
| 账号维护 | 左栏或账号菜单「账号分析」→ 更新分析 →「新写一条」回写作 |

布局冻结前重点确认：左栏常驻感、写作空态并入是否自然、选题卡切回写作的上下文、账号分析双栏、周看板回填顺序。
