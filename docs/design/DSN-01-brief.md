## Brief 正文（整段粘贴）

为 ForgeNote 设计**新用户引导态**的高保真可点击原型。视觉严格沿用我链接的本地代码（shadcn 默认 neutral：白底、近黑主按钮、浅灰边框、radius 0.625rem、Geist 字体、干净无阴影），用它现有的 Button/Card/Textarea，不要另起一套。

【产品与北极星】ForgeNote 帮单人创作者把模糊想法变成可发布的图文卡片内容包，默认中文。北极星：让创作者**第一次**用就觉得"它已经比空白 ChatGPT 更懂我的账号，而且我几乎不用解释"。

【这次只设计"新用户引导态"】终态是一个左中右底的"创作者工作台"，但新人没有资产/历史/配方，所以第一次必须轻、只做一件事：开始第一条内容任务。**不要**给新人显示内容库/配方库/历史/表现回填（对他都是空的）。

【引导态流程】
1) 首屏：顶部一句价值说明（如"你今天想做什么内容？"）；中间一个大输入框；下面 3 个可点选示例任务；并在输入前就亮出"我会替你补这些：[受众][平台][内容形式][目标][风格]"——第一屏兑现"几乎不用解释"。主按钮如"看看我怎么理解"，旁边留一句"也可以直接开始"。
2) 理解中：轻量骨架 loading。
3) "我对这次内容的理解"（核心）：AI 先给出对这次内容的账号级判断，用户确认/纠偏后再生成。**关键：要轻、可一眼扫完，不是一屏文字。**
   - 把判断压成**可扫的摘要 + 内联可改标签**，例如一句："给【刚独居·怕踩坑的年轻人】做【小红书·7张清单卡】，语气【过来人】，视觉【浅色清单】？" 每个【标签】可点改。
   - 每条判断有"依据（因为…）"和"置信度（很确定/推测/还没把握）"，但**依据默认收起，点开才看**；**只有"还没把握"的 1-2 条**才高亮/展开 +「帮我定?」。置信度的作用是**把注意力导向该纠的那 1-2 条**，不是让用户逐条读。
   - 不太确定用极克制的单色点缀（neutral 里只给这一处低饱和提示）。
   - 确认后整条收成一行带 ✓；主按钮变"都对，开始生成"。保跳过路径（认不认可都能生成）。
4) 生成内容包：占位即可，本票不设计结果区。

【要产出的状态】输入 / 理解中 / 我的理解（"有账号上下文"：贴过往帖后出现"很确定"卡、依据引用过往帖；"无上下文"：全是推测/还没把握）/ 理解失败（可重试、不丢输入）/ 认可后（收成一行）。

【冷启动】首屏给一个可选、可跳过、中等存在感的"先让我认识你的账号风格——贴一条你发过的帖"入口；贴了→依据标"来自你的账号"、置信度更高。不能挡住"只想直接输入想法"的人。

【再加一张：工作台终态示意（低保真，仅一张参照，不要做成可交互）】画一张熟练用户的工作台布局：左=账号与内容资产，中=当前内容任务与内容包，右=假设条/配方/生成控制，底=历史版本与表现回填。标注"这些区随用户积累逐块显形（账号记忆→左、首个历史→底、首个配方→右），不是一上来就满载"。

【硬约束】中文；不做图像/卡片渲染（不是 Canva）；不重设计内容包结果区；不暴露内部术语枚举（内容包/小红书笔记/卡片 Prompt 等）；M1 只认"模糊想法→内容"这一条主路径（"改写已有笔记"是另一入口，不要在示例里暗示）；最小可行，不做复杂账号建模。

【实现锚点（handoff 给 Claude Code 用）】每条假设在数据上需要 `rationale`（依据）+ `confidence`（high很确定/inferred推测/unsure还没把握）两个字段；请求新增可选 `account_post`（贴的过往帖）喂入假设推断。生成链路 model 输出需带这两字段。

【交付】引导态各状态的可点击原型 + 工作台终态一张示意 + 与现有 /forge 的差异点清单 + 打包给 Claude Code 的 handoff。
