## Brief 正文（整段粘贴）

**产品**：ForgeNote —— 帮单人内容创作者把模糊想法变成可发布的图文卡片内容包。现有代码是一个能跑的 Next.js 应用（`/forge` 工作台、登录、生成、配方）。默认中文（zh-Hans）。

**北极星（本次设计要服务的目标）**：让创作者**第一次**用就感到"它已经比空白的 ChatGPT 更懂我的账号，而且我几乎不用解释"。

**要重设计的环节（仅此一块，别动其他）**：**假设条**。现状是生成结果里一个"平台/受众/语气"的表单式默认值面板，编辑率只有 ~17%，说明没被理解、不觉得值得动。

**已定的设计方向（请在此基础上深化，不要推翻）**：
1. **把假设前移到生成前**：流程改成 `输入想法 → 「我对这次内容的理解」(假设判断步骤，先确认/纠偏) → 生成内容包`。假设成为生成前的必经一屏，而不是事后埋在结果里的面板。
2. **每条假设 = 一张"判断卡"**，含三要素：
   - AI 的判断（自然语言陈述，如"受众：刚毕业、第一次独居的年轻人"）；
   - **依据**（"因为……"，如"因为你想法里写了'第一次独居'"或"因为你过往帖多发小红书"）；
   - **置信度**（高 / 推断 / 不太确定），并据此引导纠偏：不让用户逐条审，只把注意力导向最该纠的 1–2 条；"不太确定"的卡主动求纠偏（"帮我定？"）。
   - 操作：认可 / 调整 / 不对（取代现有的 dismiss/restore）。
3. **首屏冷启动**：可选"贴一条你发过的帖，让我先认识你的账号风格"。贴了 → 判断卡依据标"来自你的账号"、置信度更高；没贴 → 退化为只从想法推断（不报错），置信度整体下调、依据标"来自你的想法"。这是"账号记忆在第一次就兑现"的最便宜方式。

**要产出的屏（原型）**：
- 屏1：首屏（想法输入 + 可选"贴过往帖"冷启动入口）。
- 屏2：「我对这次内容的理解」（4 张判断卡：受众=高、平台与形式=推断、语气=推断、视觉方向=不太确定；底部主 CTA "用这些理解生成内容包" + 次级 "重新理解"）。
- 状态与边界：理解中（轻量 loading）/ 无账号上下文（全标推断、不太确定）/ 理解失败（可重试、不丢输入）/ 认可后视觉收敛。

**硬约束（务必遵守）**：
- 复用仓库现有设计系统与组件（shadcn/Tailwind），别另起一套视觉。
- **不做图像/卡片渲染**（不是 Canva）；不重设计"内容包"结果区（那是另一张票）；不碰配方/登录/发布/日历。
- 默认中文；删 dev 口吻文案（如副标题"M1 不做自动发布"）；用户侧不要暴露内部 taxonomy（"内容包/小红书笔记/卡片 Prompt"这类枚举）。
- 最小可行：不要求复杂账号建模或自动学习。

**实现锚点（设计需为它留好接口，handoff 给 Claude Code）**：判断卡的"依据 + 置信度"不是纯前端文案——生成契约里每条 assumption 需要新增 `rationale`（依据）与 `confidence`（高/推断/不太确定）两个字段。现状 assumption 只有 value/state/source（见 `src/lib/ai/types.ts`、`src/components/forge/AssumptionPanel.tsx`、`ForgeWorkbench.tsx` 的 DEFAULT_ASSUMPTIONS）。设计请明确每条假设要展示这两个新字段。

**代码库里值得先读的文件**：
- `src/components/forge/AssumptionPanel.tsx`、`src/components/forge/ForgeWorkbench.tsx`（现状假设条 + 状态机）
- `src/components/forge/OutcomePanel.tsx`（生成结果区，本票不改但要衔接）
- `src/lib/ai/types.ts`（生成契约 / Assumption 类型）
- `src/lib/constants.ts`、`src/lib/copy/zh-Hans.ts`（产品文案 / UI 文案）
- `src/app/forge/page.tsx`、`docs/UIUX-M1.md`（现状 /forge 与 UX 规范）
