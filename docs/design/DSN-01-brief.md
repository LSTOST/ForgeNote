## 本次目标
产出 DSN-01 高保真**可点击**原型（新用户引导态 + 账号级假设），足够让 Codex 拆下一张实现票（I-20）。这次重点验证两件事：**像不像一个真 web app（不是满屏文字）**，以及**假设条像不像"看一眼方向、只改不对的"（不是逐条审稿）**。

## 视觉方向（暖编辑风，覆盖之前的冷 neutral）
ForgeNote 是**文字内容创作工作台**，视觉要像 Granola / iA Writer / Bear 那类"创作工具"的温度——暖色纸感、留白、克制，让人想停下来写；**不要** shadcn 冷灰那种 SaaS 后台气质。这条**覆盖**之前"严格匹配本地冷 neutral"的要求（产品主题 token 以后也会跟着暖化）。仍然克制：暖归暖，不花哨、不渐变、不堆装饰。

## app 外壳 + 交互态（硬要求，别只画静态屏）
交付的是"一个 app"，不是几张图。**第一天就有真 app 外壳**，壳里内容对新人才轻、区随积累逐块长（progressive disclosure）。
M1 骨架优先级：
- **必须**：持久外壳（顶栏：当前任务名 + `已自动保存 hh:mm` + 账号；左导航栏，对新人安静、大多空态）；**真实交互态**（hover/focus/active、加载骨架、乐观更新、平滑过渡）；键盘基础（Enter 提交/前进、Esc 返回、focus ring、tab 顺序）；自动保存 + 草稿持久。
- **快速跟进**：⌘K 命令面板。
- **延后（工作台期）**：多面板可拖拽、完整键盘导航、富版本历史、协作 chrome。

## Brief 正文（整段粘贴）

为 ForgeNote 设计**新用户引导态**的高保真**可点击** web app 原型（不是静态屏）。

【视觉】暖色文字编辑风（纸感、留白、克制，像 Granola / iA Writer / Bear），不要冷灰 SaaS 后台感；不花哨、不渐变。中文，Geist 或同级无衬线。

【这是个真 app，不是网页】第一天就有真 app 外壳：顶栏（当前任务名 + "已自动保存 hh:mm" + 账号）、左导航栏（对新人安静、大多空态）、真实交互态（hover/focus/active、加载骨架、乐观更新、平滑过渡）、键盘（Enter 前进 / Esc 返回 / focus ring）、自动保存草稿。壳一直在，内容对新人才轻、区随积累逐块显形。

【产品与北极星】帮单人创作者把模糊想法变成可直接做的内容方案，默认中文。北极星：第一次用就觉得"它已经比空白 ChatGPT 更懂我的账号，而且我几乎不用解释"。

【首屏：只剩一句价值 + 输入框 + 一个按钮】
- 标题（价值承诺，不解释流程）："把一个模糊想法，变成可以直接开始做的内容方案"。
- 一个大输入框，placeholder 传达"想法没整理好也能直接丢进来"。
- 一个主按钮："生成内容方案 →"。
- 全部删除：流程解释、"受众/平台/风格我会自动判断"、"先给你看我怎么理解"、"没头绪？试试…"、"已认识你的账号风格"、"也可以直接开始"、字数统计。冷启动最多压成输入框下一行小灰字链接（"想更准？贴一条你发过的帖"），可有可无。
- 首屏不教 ForgeNote 流程，只让用户明白：想法没整理好也能直接丢进来。

【方案态（原"我对这次内容的理解"，核心）：看一眼方向、只改不对的，不是逐条审稿】
- 标题不要"我对这次内容的理解"（像 AI 交卷）。用"我先帮你定了这次的方向，改不对的就行"。
- **第一行先讲实质**（最重要）：先证明 AI 读懂了想法本身。按用户输入"要不要存、存多少、放哪、何时能动"，写："这次我会替你讲清：要不要存 · 存多少 · 放在哪 · 什么时候能动"。
- 再给 3 个真正影响生成的判断，做成**内联可改标签**：受众 / 内容形式 / 表达角度（"平台"并进内容形式；"视觉"不在这一步、也不要和卡片重复出现）。
- **默认全部已定，唯一动作是"改"**：不要每项的"认可/调整/不对/帮我定"四按钮；标签点一下即可改，不需要逐项确认。
- 需要用户拿主意的那一项（原"视觉·没把握"）：① 不做成橙色描边大卡（像告警）；② 降为一行轻量提示、靠后放；③ 文案从"还没把握"改成面向收益的"这项你定一下会更贴你的账号"。不要把模型不确定变成用户负担。
- 底部只留一个前进键："生成内容方案 →"，下面一行小灰字"没改的我按默认理解来"；删掉与它并列的"跳过，直接生成"。
- 保留"它真的懂我"：实质摘要 + 账号级标签要具体（别变成空泛问卷）。

【生成内容包】占位即可，本票不设计结果区。

【要产出的状态】首屏输入 / 理解中（轻量骨架）/ 方案态（"有账号上下文"：标签更笃定、可点开看依据并引用过往帖；"无上下文"：靠后那项提示"定一下更准"）/ 理解失败（可重试、不丢输入）/ 已改若干项（"已确认 n/3，未改按默认来"）。

【再加一张：工作台终态示意（暖色、低保真、不可交互，仅一张参照）】熟练用户布局：左=账号与内容资产，中=当前内容任务与内容包，右=本次假设条/配方/生成控制，底=历史版本与表现回填。标注"新人第一次只看到中区任务 + 右区方案；其余区随账号资产/历史/配方攒起来再逐块显形，不是一上来就满载"。

【硬约束】中文；不做图像/卡片渲染（不是 Canva，工作台示意里的卡片只是占位）；不重设计内容包结果区；不暴露内部术语枚举（内容包/小红书笔记/卡片 Prompt 等）；M1 只认"模糊想法→内容"这一条主路径（"改写已有笔记"是另一入口，不在示例里暗示）；最小可行，不做复杂账号建模。

【实现锚点（handoff 给 Claude Code）】每条假设需要 `rationale`（依据）+ `confidence`（high/inferred/unsure）字段；请求新增可选 `account_post`（过往帖）喂入推断；生成链路 model 输出带这两字段。实现票还须预算 **app 外壳 + 交互层**（外壳布局、真实交互态、键盘、自动保存草稿），不是只渲染屏。

【交付】引导态各状态的**可点击**原型 + 工作台终态一张暖色示意 + 与现有 /forge 的差异点清单 + 打包给 Claude Code 的 handoff（仅设计参考，不并入源码）。
