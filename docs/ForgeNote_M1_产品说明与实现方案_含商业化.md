# ForgeNote M1 产品说明：面向知识型个人创作者的跨语言跨平台内容工作台

## 1. 产品定位

ForgeNote 是一个面向知识型个人创作者的内容创作与沉淀工作台。它帮助用户把一个尚未整理好的想法、经历、观察或观点，转化为可发布、可复用、可跨语言和跨平台延展的内容资产。

ForgeNote 不定位为爆款改写器、批量分发工具、热点搬运工具或通用聊天机器人。它的核心价值不是替用户“多写几篇”，而是减少每次创作时重复发生的判断：这件事应该讲给谁听、应该从哪个角度讲、适合做成什么内容、哪些内容结构以后还能复用。

产品主链路：

```text
模糊想法
→ 内容原型判断
→ 内容结构生成
→ 可编辑内容草稿
→ 多语言 / 多平台渲染
→ 保存为可复用内容资产
→ 发布后回填结果
→ 下一次生成更贴近账号
```

## 2. M1 目标用户

M1 只服务一类用户：

> 持续在一个或多个平台输出知识、经验、方法论或个人观察的个人创作者。

典型用户包括：

- 独立开发者、创业者、产品经理、设计师；
- 自由职业者、咨询师、研究者、教师；
- 有长期主题积累的生活方式、成长、职业、效率、知识型创作者；
- 同时面向中文和英文受众的个人品牌创作者。

他们的共同问题不是“不会写一句文案”，而是脑中有经验、有观察、有素材，但每次都要重新决定如何表达、拆成什么结构、适配哪些平台、是否值得沉淀为以后可复用的方法。

M1 不优先服务：

- 纯热点搬运、批量矩阵号；
- 商品带货、广告投放型内容生产；
- 新闻资讯重写；
- 纯娱乐内容、情绪鸡汤、无长期主题沉淀的随机发帖；
- 自动发布、多账号运营和大规模分发场景。

## 3. 内容范围：先限定五类内容原型

ForgeNote 不应让用户先选择“Narrative / Visual / Temporal”。这些是内部表达模态，不是用户语言。

用户面对的是更自然的内容原型。M1 只支持五类：

| 内容原型 | 用户常见输入 | 核心结构 | 常见输出 |
|---|---|---|---|
| 经验复盘 | “我砍掉一个功能后，发现用户真正需要的是……” | 事件 → 误判/冲突 → 证据 → 反思 → 方法 | X、LinkedIn、公众号、视频口播 |
| 知识解释 | “为什么独居的人需要备用金？” | 问题 → 原理 → 判断方法 → 行动建议 | 小红书、公众号、IG carousel、短视频 |
| 清单/指南 | “第一次独居有哪些钱一定要留？” | 场景 → 清单 → 判断标准 → 使用条件 | 小红书卡片、IG carousel、文章、视频脚本 |
| 观点表达 | “我越来越觉得很多产品功能根本不该做。” | 观点 → 反例/事实 → 推理 → 收束 | X、LinkedIn、公众号、短视频 |
| 案例拆解 | “这个产品为什么让人一直用？” | 对象 → 现象 → 拆解维度 → 可复用结论 | X thread、公众号、视频脚本、图文卡片 |

这五类是 M1 的边界。任何输入都先被归入其中一个内容原型，或由系统明确提示“当前想法不属于 M1 已支持的内容类型”。

> **M1 渲染范围注**：上表「常见输出」为完整版图。M1 实际渲染仅限 **小红书 / X / 图片 Prompt**；
> LinkedIn、公众号、IG、视频口播与短视频脚本均为 M1 后的派生方向，Temporal 模态整体延后
> （与方向文档 §4 M1 划线一致）。

## 4. 三种表达模态与多平台渲染

ForgeNote 内部保留三种表达模态，但不要求用户直接选择：

- **Narrative**：线性叙事与观点表达，用于短文、长文、thread、文章、口播稿。
- **Visual**：分卡、分屏、信息层级表达，用于小红书、Instagram carousel、知识卡、封面与图片 Prompt。
- **Temporal**：时间轴与镜头节奏表达，用于抖音、TikTok、B站、YouTube 的视频脚本与分镜 Prompt。

平台不是内容类型，而是渲染目标。系统关系必须固定为：

```text
内容原型
→ 内容结构
→ 表达模态
→ 平台渲染
```

例如：

```text
“第一次独居的备用金”
→ 清单/指南
→ 问题 → 分类清单 → 金额判断 → 动用条件
→ Visual
→ 小红书 7 张卡片 / Instagram carousel / 图片 Prompt
```

```text
“我删掉一个做了三周的功能”
→ 经验复盘
→ 事件 → 误判 → 证据 → 洞察 → 方法
→ Narrative
→ X thread / LinkedIn 帖子 / 公众号长文
```

```text
“30 天建立早起习惯”
→ 知识解释 + 清单/指南
→ 误区 → 原理 → 30 天步骤 → 常见失败点
→ Temporal
→ TikTok / 抖音口播脚本 / YouTube 视频大纲
```

（第三例为完整版图示意；Temporal 模态与视频类渲染为 M1 后能力。）

## 5. M1 核心体验

用户不需要先填完受众、平台、语气、长度、结构等复杂表单。系统应先根据用户的想法和已沉淀的账号记忆，提出一版“内容方向”。

用户在 M1 中只完成四件事：

1. 写下模糊想法；
2. 看系统是否抓对了内容原型、受众、表达角度和结构；
3. 修改不对的部分；
4. 生成一个首选平台的可编辑内容草稿，再按需要派生其他平台版本。

首选输出平台必须存在。M1 不应在第一次生成时同时产出所有平台版本，否则用户会被多输出淹没。用户先选一个“本次主输出”，其他平台在内容稳定后作为派生输出生成。

## 6. 产品中的具体实现

### 6.1 新建内容：输入与识别

新建内容页只保留一个大输入框。用户可以输入中文或英文，也可以混合输入。

输入后，系统生成一个 `Content Brief`，包含：

- 内容原型：经验复盘 / 知识解释 / 清单指南 / 观点表达 / 案例拆解；
- 主题与核心问题；
- 默认受众；
- 表达角度；
- 推荐表达模态；
- 推荐首选平台；
- 需要用户确认的不确定点。

用户看到的是自然语言，而不是 token 或 schema。系统内部可使用结构字段，但 UI 默认应显示人类可读的名称。

示例：

```text
你的想法：
“上线前一晚，我把做了三周的功能砍掉了。”

系统建议：
内容类型：经验复盘
核心角度：不是功能没做好，而是重新理解用户真实需求
适合讲给：独立开发者 / 产品创作者
推荐先做：X thread
可派生：LinkedIn 帖子、公众号长文、视频口播稿
```

### 6.2 中间工作区：可编辑内容，不是 token 编辑器

中间区域必须是当前内容的可阅读草稿，像一个真正的文字创作区，而不是 slot、token、规则表或配置面板。

内容在生成前可以显示“结构提纲”；生成后必须显示一份可编辑的半成品内容。用户可以：

- 修改标题、开头、段落、卡片；
- 选中一段后要求“换角度、补案例、写得更具体、缩短、改成更口语”；
- 调整某一张图文卡或某一段视频脚本；
- 保留原版本并生成局部新版本。

结构不是替代内容，而是驱动内容。结构应在右侧或可折叠区存在，帮助用户控制创作方向。

### 6.3 右侧：轻量结构控制；对话助手在中区底部

右侧只承担两类工作：

- 当前内容方向：内容原型、受众、表达角度、主输出、语气；
- 结构控制：当前结构的关键段落或卡片顺序。

右侧不应堆满复杂参数，不应把用户暴露给 `loss_open`、`narrative_retro` 等机器 token。内部 token 仅用于数据层、配方层与 AI 推理层；界面必须显示人类可读的标签和解释。

**对话助手位于中区底部**（默认胶囊态，点击弹出、不用即收起——v3.14 已定版），必须可快速打开，这是 ForgeNote 与传统编辑器的关键差异。用户可以输入：

```text
第二段太像产品复盘，改得更像一个人在深夜做决定。
```

```text
把这条变成给第一次独居的人看的小红书 7 张卡片。
```

```text
保留观点，但写得更适合英文独立开发者。
```

### 6.4 左侧：任务流、内容流和选题雷达

左侧不能空，也不能做成复杂 CMS。它应该承担创作者的“工作记忆”：

- 当前任务；
- 最近编辑的 3–5 条内容；
- 新建内容；
- 每周选题雷达；
- 已发布内容的快速入口。

选题雷达是“任务来源”，不是中间编辑区的一部分。它可以提供基于账号主题、历史内容和表现回填生成的选题建议。用户点击一个选题后，进入新的内容任务。

### 6.5 底部：输出、版本与沉淀

底部是动作栏，不应是第二个信息系统。只放：

- 当前版本；
- 版本历史；
- 保存为配方；
- 主输出平台；
- 派生输出入口；
- 生成内容按钮。

输出流程应是：

```text
先生成主输出
→ 用户编辑稳定
→ 再派生其他平台版本
```

例如，先生成小红书 7 张卡片，再一键派生 Instagram carousel、公众号文章大纲、短视频口播脚本和图片 Prompt。

### 6.6 配方与账号记忆

配方不能完全隐藏成后台机制。对目标用户而言，配方是“我以后还想这样做”的显式资产。

配方至少包含：

- 内容原型；
- 结构模板；
- 常用受众；
- 表达角度；
- 语气；
- 视觉规则；
- 常用输出平台；
- 已验证表现信号。

但配方不保存完整文章作为核心内容，也不等于 Prompt 模板库。它保存的是可复用的“内容生产方法”。

账号记忆也必须可查看、可关闭、可纠正。包括：

- 账号定位；
- 受众画像；
- 常用主题；
- 表达习惯；
- 视觉偏好；
- 不想再出现的表达；
- 已验证的内容规律。

## 7. M1 页面地图

```text
1. 新建内容
   模糊想法输入
   → 系统给出 Content Brief
   → 用户修正方向

2. 内容工作区
   左：任务流 / 最近内容 / 选题雷达
   中：当前内容草稿与局部编辑 + 对话助手（底部胶囊）
   右：方向与结构
   底：主输出、版本、派生、保存配方

3. 内容资产
   搜索、筛选、重跑、派生、回看表现

4. 配方库
   已验证的内容原型、结构、视觉规则与输出策略

5. 账号记忆
   账号长期上下文与可纠正的学习结果
```

## 8. M1 不做什么

M1 不做：

- 自动发布；
- 多账号批量运营；
- 复杂内容日历；
- 团队协作审批；
- 完整 Canva 式拖拽设计器；
- 爆款抓取与搬运；
- 平台数据自动抓取；
- 多模型市场；
- 所有内容类型的无限支持。

M1 的目标不是“覆盖所有创作需求”，而是验证：一个有真实经验和长期主题的个人创作者，是否愿意把日常内容创作从聊天窗口迁移到 ForgeNote，因为它更懂自己的表达、更少重复输入、更能把一次创作变成以后可复用的内容资产。

## 9. M1 验收指标

M1 只验证以下行为：

- 用户是否能在 1 分钟内从模糊想法获得第一份可编辑草稿；
- 用户是否愿意修改系统给出的 Content Brief，而不是从零重新描述；
- 用户是否会从一个主输出派生至少一个其他平台版本；
- 用户是否愿意保存一个配方；
- 用户是否在下一次创作时复用账号记忆或配方；
- 用户是否认为 ForgeNote 比空白聊天窗口更省力、更贴近自己的内容方式。

## 10. 商业化与变现路径

### 10.1 商业化判断

限定目标用户与内容边界后，ForgeNote 的商业化能力会更强。它不再卖“生成一篇文案”，而是卖持续内容生产的工作流与记忆：系统是否记住创作者是谁，是否能把一次创作沉淀为以后可复用的方法，是否能把稳定内容派生成跨语言跨平台版本，以及是否能通过历史结果让下一次生成更贴近账号。

ForgeNote 的可收费价值应集中在四类长期资产：账号记忆、配方复用、跨语言与跨平台派生、表现回填与学习。产品本质不是“AI 写作订阅”，而是“个人创作者的内容生产工作流订阅”。

### 10.2 推荐收费模型

M1 应采用“免费体验 + 个人订阅 + 使用额度”的混合模式，不宜一开始承诺无限生成，也不宜过早进入团队协作。

**免费版**的目标是让用户完成最小闭环：模糊想法 → 内容方向 → 第一份可编辑草稿 → 一次主输出。免费版可限制内容任务、派生输出、账号记忆和配方保存数量，但必须让用户体验到 ForgeNote 比空白聊天窗口更懂自己的内容方式。

**Pro：个人创作者订阅**建议初始测试 `$12–20/月` 或 `¥79–129/月`。包含账号记忆、更高内容任务额度、主输出与派生输出、中文与英文输出、配方保存与复用、内容资产检索、基础表现回填、局部重写与版本历史。定价不按字数，而是按“减少重复解释、减少重做、减少跨平台改写、持续积累内容资产”收费。

**Creator Pro / Solo Studio**面向运营多个主题、多个账号或稳定跨语言输出的重度个人创作者，建议测试 `$29–49/月`。增加多账号记忆、完整内容资产库、已验证配方、更高跨语言与跨平台派生额度、表现归因、更长版本历史，以及高级视觉配方和图片/视频 Prompt 派生。

**小团队版**应在个人版验证留存后再做，可测试 `$79–149/团队/月 + 席位或使用额度`。后续增加共享账号记忆、共享配方、审核反馈、统一品牌规则、权限与协作。M1 不做团队协作，避免过早变成企业营销后台。

### 10.3 后期非订阅收入

后期可以增加两类收入：第一是官方配方包，例如独立开发复盘、个人品牌观点表达、知识卡片、生活指南、案例拆解等经过验证的“结构 + 适用条件 + 输出策略”资产，而不是普通 Prompt 模板；第二是专业服务，例如账号记忆初始化、内容策略诊断、跨语言内容系统搭建和个人品牌内容工作流配置。

### 10.4 核心商业风险与验证问题

最大风险是：如果 ForgeNote 只能把一条内容改写成多个平台版本，用户会把它当成 ChatGPT、Claude 或其他 AI 写作工具的附属工具，付费上限会很低。真正的留存和涨价空间来自迁移成本：用户完成多次创作后，是否感觉系统已经理解自己的内容方式、能复用成功方法、能减少重复输入，并且不愿意回到空白聊天窗口重新解释自己。

M1 最优先验证的商业问题不是“用户是否愿意为多平台输出付费”，而是：**用户完成 5–10 次创作后，是否愿意继续把账号记忆、配方和内容历史留在 ForgeNote，而不是回到空白聊天窗口。**

可以用以下信号判断：是否连续创建 5 次以上内容任务；是否主动保存至少一个配方；是否在下一次创作中复用账号记忆或配方；是否从主输出派生至少一个其他平台版本；是否愿意付费保留更长历史、更高派生额度或更多账号记忆。
