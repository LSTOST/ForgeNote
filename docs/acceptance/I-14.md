# I-14 Acceptance — PostHog / Sentry 基础观测（scaffold）

> 票：补基础观测 scaffold。无外部 key 时只做 additive scaffold + 文档、no-op，不阻塞主链路、不引入大型依赖。
> **状态：Done（scaffold + 文档）。** 无 key 时 no-op、build 不失败、doctor 仅 optional 提示；真实 SDK 接入留作后续（已标 TODO）。

## 范围内

- 新增 `src/lib/observability.ts`：**零依赖、no-op** 观测 scaffold，稳定调用点：
  - `captureServerError(error, context?)` — 服务端错误上报；未配置 `SENTRY_DSN` → no-op。
  - `trackClientEvent(event, properties?)` — 前端事件；SSR / 未配置 `NEXT_PUBLIC_POSTHOG_KEY` → no-op。
  - `isServerTelemetryConfigured()` / `isClientTelemetryConfigured()`。
- `.env.example`：新增可选 `SENTRY_DSN` / `NEXT_PUBLIC_POSTHOG_KEY` / `NEXT_PUBLIC_POSTHOG_HOST`（均可选）。
- `scripts/doctor.mjs`：可选观测 env 检查，新增 `info` 级别——**未配置不计 fail/warn、不阻断**。
- 文档：`docs/DEPLOYMENT.md` 环境变量 + 启用说明 + 安全约束。

## 明确不做（守边界）

- **不引入** Sentry / PostHog SDK；不硬连任何真实服务（只 scaffold + TODO）。
- 不做完整 analytics dashboard、不做大埋点、不采集用户输入全文。
- 不要求现在就有 key；不打印 secret；不破坏 build。
- 不把观测调用强插进已完成的 I-08~I-13 业务流程（仅提供调用点，避免改完成逻辑）。

## 设计

- 依赖判断只看 env 存在性，不读值/不打印。
- 未配置 → 全部 no-op：应用照常运行、不白屏。
- 配置 env 后仍需后续票在 `observability.ts` 的 TODO 处接入真实 SDK 才会上报（只填 env 不自动外发）。
- 安全：`captureServerError` 仅上报错误消息 + 调用方给定轻量 context（不含请求体 / 用户输入全文）；`trackClientEvent` 仅接受轻量属性，调用方不得传输入全文。
