# I-16 Acceptance — output_locale 数据与生成链路支持

> 票：为 ForgeNote M1 增加 `output_locale`（目标输出语言 / 表达偏好），严格 additive，不破坏旧数据、不改 RLS、不推翻现有实现。
> **状态：Done。** 代码、migration、自动验证、匿名边界、真实登录态带 locale 写入/回填/rerun 闭环均已通过。

## 范围内

- **数据层**：migration `0002_output_locale.sql` —— `sessions` 增 `output_locale text`。
- **类型层**：`ForgeGenerationRequest.outputLocale?: string | null`。
- **API 层**：`POST /api/forge`、`POST /api/recipes/:id/rerun` 可选 `outputLocale`；写入新 session；`GET /api/sessions/:id` 返回 `outputLocale`。
- **生成链路**：`outputLocale` 非空时向生成请求追加最小输出语言/表达偏好约束。
- **UI 层**：`/forge`（ForgeWorkbench）与 `/recipes/[id]`（RecipeRerun）各增可选自由文本输入；结果 meta 区轻量展示；`/forge?session=` 预载回填。
- **文档**：DATA-SCHEMA / API-CONTRACT / DECISIONS(D-07(b) 实现结果) / PROJECT-STATUS / TICKETS / 本文件。

## migration 原则（已满足）

`supabase/migrations/0002_output_locale.sql`：

- ✅ nullable（`output_locale text`，无 `not null`）
- ✅ 无 default
- ✅ 无 enum / 无 check 约束
- ✅ 不 backfill（`add column if not exists`，旧行保持 NULL）
- ✅ 不改 RLS（沿用 0001 `auth.uid() = user_id`）
- ✅ 非破坏（`if not exists`；仅 sessions，recipes 不动）

## 设计决策

- **仅 sessions**：recipes 本票不加列——最小闭环不需要持久化 recipe 的 locale，重跑按本次请求传入（DECISIONS D-07(b) 实现结果）。
- **assumption 维度不做**：避免改 ForgeWorkbench 假设种子语义，留作后续。
- **不与国家/平台/市场绑定，非 enum，不写入 Profile，不做 UI 多语言 / 资源文件外化 / locale 下拉 / 自动翻译。**
- **非破坏保障（关键）**：route 仅在 `outputLocale` 非空时才写 `output_locale` 列（条件 spread）。因此未指定 locale 的既有 forge / rerun 流程**即使 migration 0002 尚未应用也不依赖该列、不回归**；只有使用 locale 的新路径需要该列。

## 字段 / 校验规则

- 服务端 `normalizeOutputLocale`：trim；空串视为 `null`。
- 长度上限 `MAX_OUTPUT_LOCALE_CHARS = 120`，由各 route 的 zod schema 校验，超长 → `VALIDATION_FAILED`（在鉴权前，与现有 body schema 校验一致）。
- 允许 `zh-Hans` / `en` / `en-US` / `ja` / `es` / 自由文本（如 `English for US carousel audience`）。
