> **状态：Done。** 已纳入 npm；safe-mode（无登录态 / 无模型 key → 明确 SKIP）已实测；文档化运行方式与判定语义。

## 范围内

- 新增 npm script `eval:forge`（`node scripts/eval-forge.mjs`）。
- safe-mode 语义（核心）：
  - 未设 `FORGENOTE_AUTH_COOKIE` → 打印 `SKIP` 并 **exit 0**（不计失败、不打印 secret）。
  - 模型未配置（`MODEL_NOT_CONFIGURED`）→ `SKIP` exit 0。
  - cookie 无效/过期（`AUTH_REQUIRED`）→ 明确 `FAIL` exit 1。
  - 用例检查项未过 → `FAIL` exit 1（逐项打印检查名）。
  - 全通过 → exit 0。
- `doctor` 纳入 `eval:forge` 脚本存在性检查。
- 文档：`docs/RUNBOOK.md` 增「eval gate」运行方式 + 判定语义 + 为何不进 PR CI。

## 明确不做

- 不做自动内容评分模型、不做大型样例集、不做 benchmark 平台。
- **不把真实模型调用塞进 PR CI**：eval 需真实模型 + 登录态，强行进 CI 会因缺 key/登录态无意义失败。eval 为手动 / 本地门禁；CI 仍只跑 doctor / lint / typecheck / build。

## 改动

- `scripts/eval-forge.mjs`：新增 `SkipError`；入口处无 cookie → SKIP exit 0；`MODEL_NOT_CONFIGURED` → SKIP exit 0；`AUTH_REQUIRED`（cookie 失效）→ 明确 FAIL。检查逻辑（标题/卡片/话题/关键词/禁用词/verification/intent）未改。
- `package.json`：`eval:forge`。
- `scripts/doctor.mjs`：脚本清单加入 `eval:forge`。
- `docs/RUNBOOK.md`：运行方式与判定语义。

## 安全

- 不依赖 service role。
