## 范围内（已交付）

- 新增 `src/lib/copy/`：
  - `zh-Hans.ts` —— **规范源**（current 中文文案逐字迁移）+ `Copy = typeof zhHans`。
  - `en.ts` —— `en: Copy` 英文 scaffold（编译期强制与 zh-Hans **同一组 key**）。
  - `index.ts` —— `copy`（默认 zh-Hans，组件消费）、`dictionaries`、`Locale`、`DEFAULT_LOCALE`、`getCopy(locale)`（预留）。
- 资源覆盖的稳定文案分组：`nav`（TopNav）、`login`、`forge`、`recipes`、`recipeDetail`、`profile`、`outcome`、`recipePanel`、`assumptions`。
- **已接线消费（live，证明脚手架真实可用、无 raw key）**：
  - `TopNav`：导航标签（Forge / 配方库 / 偏好）+ 退出 / 退出登录 title。
  - `/recipes` 页头：标题 / 描述 / 「新建配方」。
  - `/profile` 页头：标题 / 描述。
- 默认 zh-Hans，UI 文案与之前**逐字一致**，行为不变。

## 明确不做（守边界）

- 不做运行时 locale selector / 语言切换 UI、不做路由 locale prefix、不做 i18n middleware。
- 不存用户语言偏好、不与 `output_locale` 生成联动、不做自动翻译。
- 不引入 i18n 依赖（next-intl / react-intl 等）——纯 TS object + typed helper。
- 不改 prompt / DB / RLS / API contract；不改 Batch D（I-12/13/14）、不改 I-11 profile 行为。
- 不为抽文案重构组件结构；不强行抽临时 error/debug 文案。

## key 不漂移保证

- `Copy = typeof zhHans`（不 `as const`，值收敛为 string）；`en: Copy` → en 缺 key / 多 key / 改 key 在 `npm run typecheck` 即报错。本轮 typecheck 通过 = 两套 key 一致。

## 未抽取 / scaffold-only（增量采纳，记录清楚）

- **scaffold-only（资源已含条目、组件暂未接线）**：`login`（Google/邮箱/页脚文案）、`forge.outputLocaleLabel`、`recipeDetail`、`outcome` 各 section 标题、`recipePanel.emptyTitle`、`assumptions`。这些组件仍用原内联中文（与资源值一致），可按需小步切换为 `copy.*`，不影响当前行为。
- **有意不纳入**：`constants.ts` 已集中的产品级文案（`PRODUCT_NAME` / `SLOGAN` / `POSITIONING` / `IDEA_PLACEHOLDER` / `EXAMPLE_IDEAS`）——避免双源漂移，保持 constants 为其单一来源；动态/错误/调试/瞬时文案。
