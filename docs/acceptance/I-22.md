## Gate 1 — Product Intent

- User: logged-in solo creator using ForgeNote for the first or second time.
- Task: turn a rough idea into a publish-ready carousel / visual-card draft.
- Changed path: `/forge` result after generation.
- Loss if not done: the user can generate content, but the first result still feels like generic AI output and is not obviously worth copying or saving as a recipe.
- Out of scope: asset library, visual rendering, image generation, Canva-style editor, visual recipe system, automatic learning, publishing calendar, multi-account, DB schema/RLS changes, and `content_package` renaming.

## Implementation

- `src/lib/ai/types.ts`
  - Keeps `cardPrompts` field name for DB/API compatibility.
  - Adds optional `body` and `visualDirection` to each card prompt item.
- `src/lib/ai/generate.ts`
  - Prompt now requires 5-8 cards, each with title, card copy/body, and visual direction.
  - `recipe.acceptance` is framed as pre-publish checks.
  - Verification now checks card count, structure/copy alignment, per-card copy, per-card visual direction, hashtag range, and banned words across body/titles/comment/cards.
- `src/lib/ai/mock-generator.ts`
  - Mock output now returns 7 pages with direct card copy plus visual directions.
- `src/components/forge/OutcomePanel.tsx`
  - Result is organized around `发布正文`, `逐页卡片文案`, `配图方向`, `发布前检查`, `发布话题`, and `评论区引导`.
  - Copy action changed from visual prompt language to `复制逐页卡片文案`.
  - Session/output-locale metadata moved out of the primary visual position.
- `src/components/forge/RecipePanel.tsx`
  - Save area now explains which reusable judgments will be saved: audience, goal, tone, and structure.
- `src/lib/copy/{zh-Hans,en}.ts`
  - UI copy updated with key parity preserved.
- `docs/API-CONTRACT.md` / `docs/DATA-SCHEMA.md`
  - Documented compatible `cardPrompts.prompt/body/visualDirection` structure.

No database migration, RLS change, new dependency, route change, or schema rename.

## Gate 3 — User Path Acceptance

Required path:

1. Logged-in user opens `/forge`.
2. User enters `想做一组第一次独居备用金清单的图文卡片`.
3. User reaches direction review and edits one assumption to `新手父母`.
4. User generates successfully.
5. Result includes:
   - a publish body;
   - 5-8 page-by-page card drafts;

   - pre-publish checklist;

6. User refreshes `/forge?session=<id>` and the upgraded result remains visible.
