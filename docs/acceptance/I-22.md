## Gate 1 — Product Intent

- User: logged-in solo creator using ForgeNote for the first or second time.
- Task: turn a rough idea into a publish-ready carousel / visual-card draft.
- Changed path: `/forge` result after generation.
- Loss if not done: the user can generate content, but the first result still feels like generic AI output and is not obviously worth copying or saving as a recipe.
- Out of scope: asset library, visual rendering, image generation, Canva-style editor, visual recipe system, automatic learning, publishing calendar, multi-account, DB schema/RLS changes, and `content_package` renaming.

## Implementation

- `src/lib/ai/types.ts`
  - Keeps `cardPrompts` field name for DB/API compatibility.
  - Adds optional `body` and `visualDirection` to each card prompt item.
- `src/lib/ai/generate.ts`
  - Prompt now requires 5-8 cards, each with title, card copy/body, and visual direction.
  - `recipe.acceptance` is framed as pre-publish checks.
  - Verification now checks card count, structure/copy alignment, per-card copy, per-card visual direction, hashtag range, and banned words across body/titles/comment/cards.
- `src/lib/ai/mock-generator.ts`
  - Mock output now returns 7 pages with direct card copy plus visual directions.
- `src/components/forge/OutcomePanel.tsx`
  - Result is organized around `发布正文`, `逐页卡片文案`, `配图方向`, `发布前检查`, `发布话题`, and `评论区引导`.
  - Copy action changed from visual prompt language to `复制逐页卡片文案`.
  - Session/output-locale metadata moved out of the primary visual position.
- `src/components/forge/RecipePanel.tsx`
  - Save area now explains which reusable judgments will be saved: audience, goal, tone, and structure.
- `src/lib/copy/{zh-Hans,en}.ts`
  - UI copy updated with key parity preserved.
- `docs/API-CONTRACT.md` / `docs/DATA-SCHEMA.md`
  - Documented compatible `cardPrompts.prompt/body/visualDirection` structure.

No database migration, RLS change, new dependency, route change, or schema rename.

## Gate 3 — User Path Acceptance

Required path:

1. Logged-in user opens `/forge`.
2. User enters `想做一组第一次独居备用金清单的图文卡片`.
3. User reaches direction review and edits one assumption to `新手父母`.
4. User generates successfully.
5. Result includes:
   - a publish body;
   - 5-8 page-by-page card drafts;
   - visual direction for each page;
   - pre-publish checklist;
   - copy action for body and page-by-page card copy;
   - save-recipe area explaining reusable judgments.
6. User refreshes `/forge?session=<id>` and the upgraded result remains visible.

Current local browser check:
- Dev server: `http://127.0.0.1:3000`.
- `/forge` redirected to `/login`; local Chrome did not have a ForgeNote localhost session.
- `/login` rendered without Next.js error overlay.
- Google login click did not leave `/login` in this browser automation session; no console errors were captured.

Preview check:
- PR #10 Preview alias: `https://forge-note-git-codex-i-22-content-plan-eee8bc-lstosts-projects.vercel.app`.
- `/login` returns HTTP 200.
- `/forge` returns HTTP 307 to `/login` when unauthenticated.
- Chrome opened Preview `/forge` and landed on `/login`; the login page rendered without a Next.js error overlay.
- Attempting to click Google login from Chrome automation was blocked by browser security policy before OAuth could start.
- Owner manually opened the exact Preview URL and logged in, but OAuth returned to `https://forge-note-gold.vercel.app/forge` instead of the PR #10 Preview.
- Code check: `LoginForm` uses `window.location.origin + "/auth/callback"` for both Google OAuth `redirectTo` and Magic Link `emailRedirectTo`; there is no hard-coded production redirect in the app.
- Root cause: Supabase Auth URL Configuration does not allow the PR #10 Preview callback URL, so Supabase falls back to its configured Site URL (`forge-note-gold.vercel.app`).
- Required external config: add `https://forge-note-git-codex-i-22-content-plan-eee8bc-lstosts-projects.vercel.app/auth/callback` to Supabase Auth Redirect URLs, or add an appropriate Vercel Preview wildcard callback URL.

Status: blocked on Supabase Preview redirect configuration. After the redirect URL is allowed, Owner can log into Preview and Codex can continue the logged-in Gate 3 path from `/forge`.

## Residual Risk

- The data contract is JSON-compatible, but old sessions only have `prompt`; UI falls back to `prompt` for body/visual direction when optional fields are missing.
- Real model compliance still needs one logged-in generation sample. If the model omits `visualDirection`, deterministic verification should fail instead of silently passing.
