## Gate 1 — Product Intent

- User: logged-in solo creator who has generated an I-22 structured content plan.
- Task: save the useful result as a recipe, open that recipe, rerun it with a new input, and confirm the new output keeps the same structured content-plan shape.
- Changed path: `/forge?session=<completed>` save recipe success state.
- Loss if not done: "Save recipe" remains a dead-end confirmation. It does not prove the recipe can be reused from the detail/rerun path.
- Out of scope: asset library, visual rendering, image generation, visual recipe system, automatic learning, recipe scoring, content calendar, multi-account, DB schema/RLS changes, and `content_package` renaming.

## Implementation

- `src/components/forge/RecipePanel.tsx`
  - Stores the `recipeId` returned by `POST /api/recipes`.
  - Treats a missing `recipeId` in an otherwise successful response as a save failure.
  - Shows a `查看配方` / `View recipe` link to `/recipes/<recipeId>` after save success.
  - Adds a short reuse hint after save success so the next user action is recipe detail plus rerun, not stopping at "saved".
- `src/lib/copy/{zh-Hans,en}.ts`
  - Adds copy for the post-save recipe detail link and reuse hint.

No API change, database migration, RLS change, prompt change, or route change.
