## 3. 公开痛点证据

服务对象：H-01（灵感分散）、H-04（双语刚需）、H-05（跨平台重复劳动）。

做法：

1. 搜索：小红书 / 即刻 / X 搜「一稿多发」「内容 排版 累」「双语博主 工作流」；Reddit r/contentcreation 搜 "repurpose content tedious"。
2. 每条证据记录：链接 + 截图 + 发布时间 + 互动量。
3. 按痛点归类计数，汇总成表。

判定参考：单条痛点收集到 ≥5 条**互不相识用户**的独立抱怨，且时间跨度覆盖近半年，可作为该假设的行为证据；只找到 1–2 条的，标注「信号弱」。

纪律：所有结论标注「公开信号」，不得包装为访谈事实（USER-JOURNEY §1 的规矩）。

## 4. 异步文字问答

### 4.1 发给谁

目标 ≥5 人，回收 ≥3 份。渠道：

- 即刻 AI / 创作者圈子里正在抱怨相关痛点的人（顺着公开痛点证据找，最自然）
- X 上的华语双语博主，直接 DM
- 小红书分享创作工作流的博主
- 已有人脉 / 社群直接发

筛选底线：独立创作者、近一月有产出、双语或多平台经历满足其一。

### 4.2 话术 + 5 问（可直接复制）

> 你好，我是独立开发者，在研究内容创作者的真实工作流（不卖东西）。方便文字回答 5 个小问题吗？几分钟就好，感谢🙏
>
> ① 你最近一篇内容，从想法到发出去花了多久？中间用了哪些工具？
> ② 同一内容发多个平台的话，每个平台版本要改多久？最烦哪一步？
> ③ 做过中英两个版本吗？多久一次？怎么做的？
> ④ 你记灵感吗？记在哪？上次翻旧灵感是什么时候？
> ⑤ 你现在为哪些创作工具付费？

对方回复热情时可追问一层（「那步为什么烦？」）。全程不介绍产品；若对方主动问，答「还在调研阶段」，结束后可补一句愿不愿意将来试原型（愿意 = 强信号，记录）。

## 5. 渠道验证

1. 用现有账号发 1–2 条与方向相关的内容（如「我怎么把一条灵感变成中英两个平台的帖子」）。
2. 记录 7 天数据：曝光 / 互动 / 私信 / 关注转化，落到 `docs/research/CHANNEL-VALIDATION.md`。
3. 判定：有**真实目标用户回应**（评论提问、私信、愿意答问卷）即通过；纯点赞不算。
4. 顺手把 §4 的问答私信发给回应者——渠道验证和问答招募是同一批人。

## 附录 A · 正式访谈指南（升级路径备用）

本轮不用。留作两个场景：① 某位问答对象特别热情，愿意升级成 20–30 分钟语音；② G2 前的原型可用性测试需要面对面环节（H-03 / H-06 在那里验证）。

### 铁律（The Mom Test）

1. 前 30 分钟不介绍产品——一介绍，对方就开始配合你演戏。
2. 只问过去的具体行为（「上次 X 你怎么做的」），不问未来意愿（「你会用吗」的回答一律无效）。
3. 恭维不算证据；主动追问、愿试原型、说出正在付费的工具才算。
4. 让对方打开电脑边翻边讲，看真实工具和文件，不听转述。

### 流程（45 分钟）

1. **暖场（5 min）**：独立开发者研究创作者工作流；无标准答案；录音许可。
2. **现状复盘（10 min）**：「拿你最近发的那篇，从想法到发出去完整走一遍」——工具链、每步耗时、发了几个平台、每个版本改了什么（H-01 / H-05）。
3. **深挖（20 min）**：
   - 灵感：「上次记了没写成的灵感，现在找给我看看」「多久记一次、记在几个地方」（H-01 / H-02）
   - AI 与结构：「打开你和 AI 的最近对话」「有没有整段推翻的经历，为什么」「想先要大纲还是直接成文」（H-03，意愿题只作线索）
   - 双语：「最近一次中英两版是什么时候、流程、月频」（H-04）
   - 复用：「有没有总结过自己的套路？存哪了？第二次真用过吗」（H-06 / H-07）
   - 付费：「为哪些创作工具付费、每月多少、最近一次为新工具掏钱是因为什么」
4. **概念反应（5 min）**：一句话切口描述，只问「解决了你说的哪个问题？**哪个没解决？**」「哪一步你最不愿意交给它？」
5. **收尾（5 min）**：「有什么我该问没问的」＋愿否试原型＋推荐 1–2 位同类创作者。
