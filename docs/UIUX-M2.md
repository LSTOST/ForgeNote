## 1. 设计原则

1. **内容工作台感**：ForgeNote 是写作/内容创作工作台，非冷 enterprise dashboard。
2. **可读可编辑**：中区展示的是人类可读内容（正文/卡片/脚本），非 token/JSON。
3. **结构幕后化**：结构生成是幕后引擎，UI 不直接暴露 raw token。控制面板用人类语言。
4. **渐进式操作**：先分析账号 → 找到可写方向 → 生成内容框架 → 生成正文 → 编辑 → 生成平台版本。
5. **60 秒判断差**：打开产品后，AI 对账号的判断应明显深于空白 ChatGPT。

## 3. /workspace 四区工作台

Owner 2026-07-06 定稿：中区必须是可读可编辑内容，结构是幕后引擎。Owner 2026-07-09 补充：用户界面使用人话，不展示内部术语；代码/API 字段可以保留内部 key。

```
┌──────────────────────────────────────────────────────────────┐
│ Top Bar: Home / 左侧面板切换 / 当前内容标题                    │
├───────┬──────────────────────────────┬───────────────────────┤
│ Left  │ Center (flex-1)               │ Right (296px)         │
│ Panel │                               │                       │
│ 252px │ Stage 0: 内容想法 + 生成按钮   │ 内容类型 + 表达形式   │
│       │ Stage A: 内容框架              │ 写作顺序              │
│       │ Stage B: 可编辑正文            │ 还差一步              │
│       │  - 每段：标题 input + 正文     │ 待裁决项              │
│       │    textarea                   │ 输出语言              │
│       │                               │ 账号分析摘要          │
│       ├───────────────────────────────┤                       │
│       │                               │                       │
├───────┴───────────────────────────────┴───────────────────────┤
│ Bottom Bar: 保存内容框架/保存这套写法 | 平台选择器 | 生成平台版本 │
└──────────────────────────────────────────────────────────────┘
```

### 3.1 Zone 1: Left Panel（252px）

- "新写一条" 按钮
- 当前账号：显示账号分析状态
- 当前内容：显示内容类型、阶段、完成度
- 本周可写选题：进入 `/radar`
- 最近内容：保留轻量占位
- 用户头像 + 退出

### 3.2 Zone 2: Center（主内容区）

三阶段渐进：

| Stage | 显示内容 | 操作 |
|-------|----------|------|
| **0 · 冷启动** | Textarea（placeholder: "写一个内容想法..."）；右栏和底栏常驻 | 输入想法 → 底栏点击生成 |
| **A · 内容框架** | 方向描述 + 分段卡片。每张卡包含段落名称、表达方式、一句可读内容、为什么这样写 | 确认框架 → 生成正文 |
| **B · 正文** | 每段可编辑：heading input + text textarea | 自由修改标题和正文 |

双向 hover 高亮：当鼠标悬停在中区某段或右栏对应段落时，对应元素高亮联动。

### 3.3 Zone 3: Right Panel（296px）

人类可读控制面板，非 raw token slider：

- 右栏在冷启动阶段也常驻展示，不因还没有内容框架而消失。
- **当前状态**：显示当前生成阶段
- **内容类型**：显示当前内容类型和表达形式
- **写作顺序**：列出每个段落，可点击选择表达方式（下拉菜单）
- **还差一步**：当内容框架未确认时显示具体原因
- **待确认**：AI 不确定的选项，提供按钮选择
- **输出语言**：预设 + 自由文本输入
- **账号分析摘要**：audience / voice / rules / proven patterns 记忆

### 3.4 Zone 4: Bottom Bar

- 底栏在冷启动阶段也常驻展示，保持工作台形态。
- 冷启动阶段："保存内容框架" disabled + 平台选择器 + "生成内容框架"
- 内容框架阶段："保存内容框架"（占位，当前 disabled）
- 正文阶段："保存这套写法"（占位，当前 disabled）
- 平台选择器：Xiaohongshu / X-Thread / Image Prompt
- 内容框架阶段："生成正文"
- 正文阶段："生成小红书版本" / "生成 X 版本" / "生成图片提示词"
- 平台版本以 overlay 展示

## 4. 交互规则

### 4.1 段落表达方式编辑
- 点击右栏中的段落 → 显示表达方式下拉选项
- 选择 strategy → PATCH `/api/structure/[id]/slot`
- 结构即时更新，stability 重新计算

### 4.2 待确认项
- 点击未解决的 decision → 选择选项
- 点击确认 → PATCH `/api/structure/[id]/decision`
- decision 状态变更为 user_resolved

### 4.3 双向 hover 高亮
- hover 中区某段或右栏某段 → 通过 `hoverSlot` state 匹配 `slotKey` 实现双区高亮

### 4.4 平台版本流程
```
内容框架 → 生成正文 → 编辑 sections → 选平台 → 生成平台版本
```
平台版本调用 `POST /api/content/derive` 传入用户编辑后的 sections。

## 5. /radar 本周可写选题

- 基于 account memory 生成每周选题卡片
- 每张卡片：topic / 原型建议 / 钩子示例 / 推荐平台 / 证据来源
- 点击卡片 → 跳转 `/workspace?idea=xxx` 预填想法

## 6. /first-run App Home

- 登录后的产品首页，不是账号分析页。
- 左侧轻导航：ForgeNote、+ 新写一条、账号分析、本周可写选题、常用写法、搜索、最近内容。
- 主区大留白，中央标题："今天想写哪条内容？"
- 中央大输入框 placeholder："描述一个想法、选题，或你卡住的地方"。
- 输入内容并提交 → `/workspace?idea=<encoded input>`。
- 快捷入口：找选题、内容框架、小红书版本、X 版本、图片提示词。
- 点击左侧账号分析 → `/account`；点击找选题 → `/radar`；点击新写一条 → 清空并聚焦输入框。
- 左上 ForgeNote 网站 logo → `/` 官网；工作台左上 Home → `/first-run` App Home。

## 6.1 /account 账号分析

- 粘贴账号 profile + 近期帖子
- 触发 account intake → 生成账号分析
- 引导进入 `/workspace` 开始第一个内容

## 6.2 /login 登录页

- 桌面端采用左右五五分栏：左侧为 ForgeNote 品牌视觉区，右侧为极简登录表单。
- 左侧视觉使用按桌面左栏比例裁切的克制插画：光从一道橙色门中透出，小人面向门准备进入；不放产品 Logo、文案、编辑器 mockup 或能力标签，不用额外遮罩拼接画面。
- 右侧不在登录框上方重复放产品 Logo，优先显示 Google OAuth / 微信占位，再显示邮箱密码登录。
- 微信登录仅作为灰态 disabled 占位展示，文案为"使用 微信 登录"，使用微信图标，不展示"即将支持"，不调用未接入的 Auth 能力。
- 移动端隐藏左侧品牌视觉，只显示登录表单，避免横向滚动。

## 7. 视觉方向

```
关键词：干净、克制、内容工作台、可长期使用
背景：浅灰 / 白色
主色：中性黑 / 深灰
强调色：少量暖橙或蓝色
卡片：圆角、轻边框、低阴影
字体：系统字体
避免：大面积渐变、营销海报感、冷 enterprise dashboard 感
```

### 必须避免
- 冷 enterprise dashboard 感（多面板、大量空态区）
- 过多 technical control（raw token slider）
- 重复 avatar 区域
- 无上下文的突兀按钮
- `/forge` 时代的 content_package / assumption chips 视觉遗产
