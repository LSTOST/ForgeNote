# ForgeNote UIUX · 单工作台外壳

> 版本：G0S-12 / v2.0  | 日期：2026-07-13  | 状态：implemented

## 1. 产品布局

登录后使用一个 Next.js 共享 layout：252px 可收起左栏 + 主区。路由切换只替换主区，左栏不重载；所有 URL、`?idea=` 与 `?taskId=` 深链保留。

| 视图 | 路由 | 主区职责 |
|---|---|---|
| 写作 | `/workspace` | 从想法、框架、正文到小红书产物 |
| 选题 | `/radar` | 生成并选择本周候选选题 |
| 账号分析 | `/account` | 维护影响生成的账号判断 |
| 周看板 | `/gate0` | 查看 Gate 0 使用统计并回填 fallback |
| 登录 | `/login` | 外壳外唯一独立产品页 |

`/first-run` 重定向 `/workspace`。`/profile` 路由和代码保留，但产品 UI 不提供入口。

## 2. 持久外壳

左栏从上至下：

- Home：进入 `/workspace`
- 收起左侧工作区；收起后显示 fixed「展开」按钮
- 「新写一条」
- 最近内容：最多 8 条，点击以 `/workspace?taskId=` 恢复
- 「本周可写选题」
- 辅助组：「周看板」「账号分析」
- 页脚账号菜单：「账号分析」「退出登录」

app 外壳不显示 logo，不使用旧 TopNav。BrandMark 只属于登录与营销站。

## 3. 写作视图

### 3.1 空状态

无进行中内容时，主区显示「今天想写哪条内容？」、想法输入和「开始创作」。这取代原 `/first-run` hub。

### 3.2 工作流态

```text
Stage 0 想法 → [开始创作 / 生成内容框架]
Stage A 内容框架 → [生成正文]
Stage B 可编辑正文 → 产出区：小红书 → [生成文案 + 卡片提示词] → 版本记录
```

- 中区 max-width 760px：内容与主操作。
- 右栏 296px：唯一状态 Badge、写作顺序、待确认、账号分析摘要。
- 每阶段只有一个实心 brand 主操作；平台产物记录常驻中区。
- 正文 1.2 秒防抖保存，保存状态跟随内容方向行显示。
- 生成失败保留已有输入和中间结果；未持久化明确提示刷新风险。

## 4. 选题视图

`/radar` 在外壳主区内生成账号相关选题。卡片主操作「用这条开始创作」进入 `/workspace?idea=`，只预填，不自动生成。空态允许直接从左栏新写一条。

## 5. 账号分析视图

`/account` 在外壳主区内展示资料输入与当前判断。主操作「更新账号分析」；说明明确这些判断影响内容框架、正文和小红书版本。更新失败时原资料与上一版判断继续保留。

## 6. 周看板视图

`/gate0` 在外壳主区内展示本周真实任务、发布、选题采用、fallback、结构编辑、renderer、复制和表现回填数据，并保留 fallback 回填表单。周看板是左栏辅助入口，不承担内容编辑。

## 7. 登录

`/login` 位于共享外壳之外，保持桌面左侧静态品牌视觉 + 右侧表单；移动端隐藏视觉。登录成功进入 `/workspace`。Ember 已移除。BrandMark 仅在登录视觉和营销站出现。

## 8. 视觉与文案

- 暖纸白色板保持不变。
- 圆角仅 `8 / 12 / 16 / 20px`；阴影仅使用暖褐 token。
- Owner 选择字体 A：衬线标题 + 无衬线正文。
- UI 使用「」；中文与英文/数字之间保留空格；不直呼使用者；不暴露内部字段。
- 禁止渐变、玻璃拟态、假快捷键、死按钮、重复信息、卡中卡和主操作脱离内容流。
