# ForgeNote UIUX-M1
> 范围：M1 页面、组件、状态、按钮与交互规则。

## 1. 设计原则

1. 工具感优先，不做营销页风格。
2. 页面要让用户一眼知道下一步该做什么。
3. 生成过程要可见，不能让用户觉得黑箱。
4. 假设条是核心交互，必须清楚、可改、可删。
5. 内容结果和内容配方并列展示，强调“产出 + 复用”。
6. 移动端可用，但 M1 优先桌面端。

## 6. 假设条组件

### 6.1 展示形式

```text
[平台：小红书] [内容形式：7 张卡片] [受众：第一次独居的人] [语气：成熟、清楚、不焦虑]
```

### 6.2 Chip 字段

| 字段 | 说明 |
|---|---|
| label | 维度名，例如“语气” |
| value | 当前值 |
| source | inferred / profile / edited |
| state | default / edited / dismissed |

### 6.3 状态

| 状态 | UI |
|---|---|
| default | 普通 chip |
| edited | 高亮边框，显示“已改” |
| profile | 显示“来自偏好” |
| dismissed | 不展示在主列表，进入“已删除假设”折叠区 |
| editing | chip 变成输入框 |

### 6.4 操作

| 操作 | 结果 |
|---|---|
| 点击 chip | 进入编辑态 |
| 回车 / 保存 | 状态变为 edited |
| 点击 × | 状态变为 dismissed |
| 点击恢复 | 恢复 default |
| 点击全部恢复 | 所有 chip 回到 default |
| 点击应用修改并重新生成 | 用当前假设重跑生成 |

M1 不做自动防抖生成，必须用户点击“应用修改并重新生成”。

## 7. Outcome 生成结果区

### 7.1 空态

标题：生成结果会出现在这里  
说明：输入想法并点击“开始锻造”后，将生成标题、正文、卡片 Prompt 和话题。

### 7.2 加载态

- 显示 loading skeleton
- 文案：正在锻造内容...
- 按钮禁用：复制、保存配方

### 7.3 成功态

展示结构：

```markdown
# 内容定位
# 标题备选
# 小红书正文
# 卡片结构
# 卡片 Prompt
# 发布话题
# 评论区引导
```

### 7.4 错误态

文案：生成失败，请重试。  
按钮：重新生成。

失败时当前 Session 以草稿保存（status=draft，outcome=null，error_code=GENERATION_FAILED），输入与假设不丢失，用户可稍后重试（见 DECISIONS D-04）。

### 7.5 操作按钮

| 按钮 | 规则 |
|---|---|
| 复制全文 | 复制所有 outcome |
| 复制正文 | 只复制小红书正文 |
| 复制卡片 Prompt | 只复制卡片 Prompt |
| 重新生成 | 保留输入和假设，重新生成 |
| 新建 | 清空当前 session |

### 9.1 字段

- 配方名称 input

- 取消按钮
- 保存按钮

### 9.2 状态

| 状态 | 规则 |
|---|---|

## 14. 视觉方向

M1 视觉关键词：

```text
干净、克制、工具感、内容工作台、可长期使用
```

建议：

- 背景：浅灰或白色
- 主色：中性黑 / 深灰
- 强调色：少量暖橙或蓝色
- 卡片：圆角、轻边框、低阴影
- 字体：系统字体
- 不使用大面积渐变
- 不做营销海报感
